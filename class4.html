Review: Div Element

Divs group other HTML elements together.

IDs and classes are often added to divs to apply CSS styles to a specific group of HTML elements

Div Example

http://gdicbus.github.io

Example: Div and Span Elements

HTML code:

<div class="sponsors">
  <p>
    This is a paragraph all about our sponsors. One of our 
    favorite companies of all time is 
    <span class="company-name">Neo</span>. 
    We also have some other really fantastic sponsors, 
    like <span class="company-name">Company #2</span>
    and <span class="company-name">Company #3</span>.
  </p>
</div>

CSS:

.company-name {
  color: purple;
  font-weight: bold;
}

Example Output

On the website, the paragraph would look like:

This is a paragraph all about our sponsors. One of our favorite companies of all time is Neo. We also have some other really fantastic sponsors, like Company #2 and Company #3.

IDs and Classes

  • You can name your classes and IDs anything you want.
  • A class you've created can be used many times throughout a page.
  • An ID can only be used once per page.
  • Classes and IDs add meaning and allow for easier styling.

Relative Positioning

  • Allows top, left, right, and/or bottom properties to be applied in order to remove the element from the normal flow.
  • The top/left/right/bottom properties specify how to position the element relative to where it would normally be placed.
.relative{
  position: relative; 
  left: 80px; 
  top: 20px;
  height: 100px; 
  background-color: yellow;
}
Hello, hi!

Absolute Positioning

Absolute positioning takes the element out of the normal flow and positions it in relation to the window (or the closest relative element).

.top{
  position: absolute; 
  top: 0; 
  right: 10px; 
  background-color: yellow
}

.bottom{
  position: absolute; 
  bottom: 0; 
  left:60px; 
  background-color: green
}

Down here

Absolute Positioning

  • Positions the element outside of the normal flow.
  • Other elements act as if the absolutely positioned element is not there.
  • Relative positioning and absolute positioning are used together.
  • The absolutely positioned element is offset from either 1) the <html> element, or 2) the nearest relative element.

Demo: Absolute Positioning

Real-life use case: adding a caption to an image.


Z-index

When you use positioning to move elements out of the normal flow of content, elements can overlap. Change the order of overlapping with z-index.

The element with the highest z-index goes on top.

.bottom{
  position: absolute; 
  bottom: 10px; 
  left:60px; 
  background-color: yellow;
}

.top{
  position: absolute; 
  bottom: 15px; 
  left:60px; 
  background-color: green; 
  z-index: 2;
}

Float

  • The element remains within the normal flow but moves as far to the left or right of the containing element as possible.
  • Any other elements, such as paragraphs or lists, will wrap around the floated element.
  • Always specify a width when floating an element, otherwise the element is likely to take up the whole page and not appear floated.
  • You can specify which side an element ï¬‚oats on.

Float: Example

Below a div containing an image and caption is floated to the right, allowing text to wrap around it.

Float

.float {
  float: left;
  width: 200px;
  background: yellow;

Float: Placing Elements Side By Side

If you want two block level elements to be side by side, you need to float both elements. One left, and one right.

Clear

  • Clear tells the element on which side (right, left, both) other elements cannot appear.
  • If you had an image floated left, and you did not want paragraphs to appear next to it, you would add clear: left; to the paragraph.
p {
  clear: left;
}

Example: Clear

img {
  float:left;
}

p {
  clear:left
}

Putting your site on the web

  1. Register a domain.
  2. Get (server) hosting for that domain.
  3. Upload your HTML, CSS, image files, etc. to the server.


OR use Github Pages for free: Github Pages.

Domains

  • This is a domain: http://www.something.com
  • Generally, fairly cheap (under $10/yr).
  • Can get it through the same company as your hosting, or a separate company. One example: namecheap.com.

Hosting

  • Once you have a domain, you need a hosting service to host it.
  • A hosting service provides you with a server.
  • The server is what makes your site viewable from any browser, on any network, anywhere.
  • Recommend: http://www.asmallorange.com ($5/mo)
